「Webコンテンツ作成」の成果

本テーマについて

先行する情報工学の専門科目の基礎知識を応用して実践的な能力を修得することが目的。
到達目標は次の4つ。

  • 手順書を読み、実験を行い、レポートやプレゼンテーションにより報告するという実験の一連の流れを実践できる。
  • コミュニケーション能力の重要性を理解し、グループワークを円滑に実施できる。
  • モーション解析・編集ができる。
  • HTML等を用いたWebページ作成ができる。

プロジェクト課題について

6回の講義を通じて取り組む課題。
WWW のサーバへ検索し，検索結果を表示するページを作成。

  • 検索結果をすべてページに表示する。
  • HTML ではリスト要素，テーブル要素，図などの要素を使うこと。
  • 各検索結果のデータ項目は全てを表示する必要はなく，
    一部だけ表示するのでも良い。

データは次の三種類として，そのいずれかのデータを検索，結果表示できるようにする。

  • NHK の番組表
  • グルメ情報
  • 世界の天気

HTML、CSS、JavaScriptを活用して課題を完成させる。

私はグルメ情報を選択した。

各授業回の学習事項

第1回 WWW と HTML

ブロック要素やインライン要素などの要素、そしてその要素に追加する付加情報の属性。
画像を埋め込む方法やハイパーリンクについて。
HTMl文書全体の構成についての説明と、その際に使用する各要素についてとHTMLのデバッグについて学んだ。

演習の成果:演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

要素それぞれの見た目の設定（装飾）について。
ページを装飾する記述は CSS と言い、CSS は要素の見た目を変更するもので、HTMLが最初に与えられて、次にその装飾をする CSS を記述する。
CSSをHTMLに適用する方法やCSSの書式について学んだ。

演習の成果:練習問題2-3:ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

インタプリタ言語とコンパイル言語についての説明
JavaScriptをページに追加する方法。
コンソールに出力するプログラムの作成。
JavaScriptの構文とJavaの構文の比較。
JavaScriptのデバッグについて学んだ。

演習の成果:練習問題3-6:オブジェクトの参照

第4回 DOM と JavaScript

DOMという構造について。
要素の検索、テキストの変更、CSSプロパティの変更、新しい要素の追加方法、要素の削除の方法について学んだ。
DOMを使う理由の解説。
DOMを利用した演習問題を中心的に授業を行なった。

演習の成果:練習問題4-2~4-5

第5回 イベント処理

イベントについての説明と例の提示。
イベント処理の簡単な例の解説やイベントハンドラの登録について。
入力ホームのページの例や実現方法についての説明。
input要素の種類の解説。
イベント処理のデバッグ方法について学んだ。

工夫点,苦心点,強調点,感想

今回の授業で最も苦労した点は第4回で学んだDOMとJavaScriptだ。
要素の検索やテキストの変更等に関して理解するまでにかなり時間がかかってしまった。
特に新しい要素の追加が個人的に難しく、その授業時間内では活用ができるようになるまで至らなかった。
しかし、時間をかけたのもあり、それらを扱えるようになった時はとても嬉しかった。

数当てゲームとプロジェクト課題、両方とも制作に苦労したが、 制作の自由度が高いプロジェクト課題よりも数当てゲームの方が躓いた回数が多かったように感じた。
数当てゲームでは、ボタン、ラベルの配置や入力した数字をきちんと数字として反映させるのに苦労し、
プロジェクト課題では、2回目以降の入力の際に前回の結果を削除するのが中々上手くできず難航した。
最後の授業で行ったデータ通信では、送られてきたデータを上手く表示することができなくて苦労した。
両方とも制作の過程で躓くことが多々あったので、制作が終わった時の達成感が凄まじかった。

できるだけプログラムの量を減らすように工夫したつもりだが、まだまだ削れる部分が多くあったのでそれは今後の課題だと思う。

ページのデザインについては、できるだけ同系色を選ぶことでページ内でまとまりが出るようにした。
ごちゃごちゃした感じにならず、あまりデザイン等が得意ではないにしては上手くできたほうだと思う。

ホームページはいつか作ってみたいという気持ちがあったので、その基礎のようなものを学ぶことができたのはとても良い経験になった。
今回の授業で学んだ内容だけでもできることが案外多く驚いた。
構文を覚えること自体はそこまで難しくはなかったが、それらをしっかりと活用できるようになるのには時間を要することが多かった。
CSSでページを装飾するのが個人的にかなり楽しかった。
今までプログラミング等を学んでる中では堅苦しいものが多かったので、構文を覚えるのもそれをどのように活用してページを装飾するのかを 考えるのも楽しかった。
躓いた回数が多いのは数当てゲームの方だが、長時間悩んだのはプロジェクト課題の方だった。
データを配列を利用してページに表示するところで配列を どのように表記すればいいのか分からずに悩んだ際には、二、三日上手くできる方法が思い浮かばなかった。
全体を通して見ると、苦労したり悩んでいたりした時間がかなり多かったように感じたが、 この授業ではとてもいい学びを得ることができたと思う。
新しいことを学んで技術を習得することはとても楽しいことだと、この授業を受けて改めて思った。
